# Draw_Inspect

Сервис для анализа чертежей и технической документации с использованием мультимодальных LLM (Ollama).

## Описание

Draw_Inspect — это веб-приложение, которое позволяет загружать PDF-файлы с чертежами, автоматически определять страницы с чертежами и анализировать их содержимое с помощью локальной LLM (Ollama). Приложение извлекает только страницы с чертежами, игнорируя текстовые документы и спецификации.

## Возможности

- 📄 **Автоматическое определение чертежей** — извлечение только страниц с чертежами из PDF
- 🖼️ **Поддержка изображений** — загрузка чертежей в формате изображений
- 🤖 **AI-анализ** — ответы на вопросы по содержимому чертежей с использованием VLM
- 🔒 **Локальная работа** — все вычисления выполняются локально через Ollama
- 🐳 **Docker-поддержка** — быстрый запуск в контейнере

## Требования

- Python 3.10+
- Ollama с установленными моделями:
  - `gemma4:31b` (или другая VLM модель)
  - `gemma3:27b` (или другая модель для валидации)
- Для работы с PDF: `poppler-utils`

## Установка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd Draw_Inspect
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Установка системных пакетов (для Linux)

```bash
sudo apt-get update && sudo apt-get install -y libgl1 libglib2.0-0 poppler-utils
```

## Настройка

Переменные окружения (настраиваются в `start-dev.sh` или `.env`):

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `UPLOAD_FOLDER` | Папка для загруженных файлов | `./uploads` |
| `OUTPUT_FOLDER` | Папка для результатов | `./outputs` |
| `OLLAMA_BASE_URL` | URL Ollama API | `http://localhost:11434` |
| `DRAWING_VLM_MODEL` | Модель для анализа чертежей | `gemma4:31b` |
| `DRAWING_VALIDATION_MODEL` | Модель для валидации | `gemma3:27b` |
| `DRAWING_MIN_SIZE_CM` | Минимальный размер чертежа (см) | `42.0` |
| `MAX_UPLOAD_MB` | Максимальный размер загрузки (МБ) | `1024` |

## Запуск

### Режим разработки

```bash
chmod +x start-dev.sh
./start-dev.sh
```

Приложение будет доступно по адресу: http://localhost:6002

### Docker

```bash
docker-compose up --build
```

Приложение будет доступно по адресу: http://localhost:6002

### Через SSH туннель

Для доступа к удалённому экземпляру:

```bash
chmod +x tunnel.sh
./tunnel.sh
```

Затем откройте: http://localhost:8081

## API

### Загрузка файлов

**POST** `/api/upload`

Загружает файлы (PDF или изображения) для анализа.

**Ответ:**
```json
{
  "session_id": "uuid",
  "total_pages": 5
}
```

### Анализ чертежей

**POST** `/api/analyze`

Анализирует загруженные чертежи с заданным вопросом.

**Тело запроса:**
```json
{
  "session_id": "uuid",
  "question": "Опиши основные размеры на чертеже"
}
```

**Ответ:**
```json
{
  "status": "completed",
  "response": "На чертеже представлены..."
}
```

## Использование

1. Откройте приложение в браузере
2. Загрузите PDF-файл с чертежами или изображения
3. Система автоматически определит и извлечёт страницы с чертежами
4. Введите вопрос по содержимому чертежей
5. Получите AI-ответ на основе анализа
