# Сервис анализа чертежей зданий для пожарной безопасности

Сервис выполняет анализ архитектурных чертежей (планов зданий) и отвечает на вопросы по пожарной безопасности.

## Структура проекта

```
/workspace
├── backend/
│   └── app.py              # Flask сервер (основное приложение)
├── scripts/
│   ├── __init__.py
│   ├── pdf_processor.py    # Обработка PDF (извлечение страниц в изображения)
│   ├── classifier.py       # Классификация чертежей (VLM)
│   ├── vlm_analyzer.py     # Анализ чертежей с помощью VLM
│   └── llm_responder.py    # Генерация структурированных ответов (LLM)
├── frontend/
│   └── index.html          # Одностраничный интерфейс
├── uploads/                # Папка для загруженных файлов
├── requirements.txt        # Зависимости Python
└── README.md              # Этот файл
```

## Функционал

1. **Загрузка документов** - поддержка PDF и изображений (PNG, JPG)
2. **Автоматическая классификация** чертежей:
   - Жилой поэтажный план
   - Технический этаж
   - Паркинг/подземный этаж
   - Вид здания сбоку
   - Генплан
   - Разрез
3. **Анализ с помощью VLM** (Vision Language Model) - извлечение информации из чертежей
4. **Структурированные ответы** с указанием страниц, где найдена информация

## Примеры вопросов

- Сколько лестниц на жилых этажах?
- Какова площадь квартир на этажах?
- Сколько парковочных мест?
- Какова ширина марша у лестниц?
- Какова ширина коридора?

## Установка и запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка API ключа (опционально)

Для работы с реальными VLM/LLM моделями установите переменную окружения:

```bash
export OPENAI_API_KEY="your-api-key-here"
```

Без ключа сервис будет работать в демо-режиме с тестовыми ответами.

### 3. Запуск сервера

```bash
cd backend
python app.py
```

Сервер запустится на `http://localhost:5000`

## Как это работает

1. Пользователь загружает PDF или изображения чертежей
2. Backend извлекает страницы из PDF в изображения
3. Каждый чертеж классифицируется с помощью VLM
4. При получении вопроса от пользователя:
   - VLM анализирует каждый чертеж в контексте вопроса
   - LLM генерирует структурированный ответ с привязкой к страницам
5. Ответ отображается на фронтенде с указанием конкретных страниц

## API Endpoints

- `GET /` - Главная страница (фронтенд)
- `POST /api/upload` - Загрузка файлов
  - Возвращает `session_id` и классификации чертежей
- `POST /api/analyze` - Анализ чертежей с вопросом
  - Принимает `session_id` и `question`
  - Возвращает структурированный ответ с находками и ссылками на страницы

## Технологии

- **Backend**: Flask (Python)
- **Обработка PDF**: PyMuPDF
- **VLM/LLM**: OpenAI GPT-4o-mini (или совместимый API)
- **Frontend**: HTML/CSS/JavaScript (ванильный)
